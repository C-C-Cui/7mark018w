Because output is probabilistic, always log: model name, model version, temperature, and exact prompt. Lower temperature → more reproducible. Record all in your GenAI Session Log.
01
Tokenisation
BPE sub-word splitting
Text is split into tokens — sub-word units (whole words, word fragments, or punctuation). Each token is assigned a unique integer ID from the model's vocabulary (typically 30k–100k entries). Note how long marketing terms often split into two or more tokens.
02
Embedding
High-dimensional vector space
Each token ID is mapped to a vector — typically 768–4096 numbers. Semantically related tokens occupy nearby positions. The bars show a 12-dimension slice; real embeddings have hundreds of dimensions per token.
03
Self-Attention
Contextual weighting
Each token attends to every preceding token with a learned weight (0 → 1). Darker cells = stronger influence. Future tokens are masked (—) — the model cannot "see ahead". Hover a cell to read the exact weight.
04
Next-Token Prediction
Softmax probability distribution
After attention, a feed-forward network and softmax layer produce a probability over the entire vocabulary. The model samples from this distribution — controlled by temperature. Drag the slider to see how temperature reshapes the distribution.
